## Contribution workflow

<Steps>
  <Step title="Fork the repository">
    Fork the relevant Catppuccin repository to your own GitHub account.
  </Step>
  <Step title="Create a topic branch">
    Create a [topic branch](https://git-scm.com/book/en/v2/Git-Branching-Branching-Workflows#_topic_branch) on your fork for your specific change. Do not work directly on `main`.
  </Step>
  <Step title="Make your changes">
    Apply your changes. Use the `.editorconfig` file at the root of the project to maintain consistent coding styles across editors. See [EditorConfig's website](https://editorconfig.org/) for setup instructions.
  </Step>
  <Step title="Write your commit message">
    Write a clear and explicit commit message. Following [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) is strongly recommended.

    <Tip>
      Conventional Commits give your history a consistent, machine-readable structure. Use prefixes like `feat:`, `fix:`, `docs:`, and `chore:` to make your intent clear.
    </Tip>
  </Step>
  <Step title="Open a pull request">
    Push your branch and open a pull request against the main repository. Describe what you changed and why.
  </Step>
</Steps>

## Code style

Use the `.editorconfig` file located at the root of the project to maintain consistent coding styles. This handles indentation, line endings, and other formatting details automatically in most editors.

## Commit messages

<Tip>
  Consider following [conventionalcommits.org](https://www.conventionalcommits.org/en/v1.0.0/) rules for creating explicit and meaningful commit messages.
</Tip>

Conventional Commits format:

```
<type>[optional scope]: <description>

[optional body]
```

Common types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`.

## First-time contributors

If this is your first time contributing to an open source project, the [first-contributions](https://github.com/firstcontributions/first-contributions) repository on GitHub is an excellent hands-on resource. It walks you through the GitHub workflow — forking, branching, committing, and opening a pull request — in a safe environment.

Feel free to join the [Catppuccin Discord](https://discord.com/servers/catppuccin-907385605422448742) to ask questions and clarify your understanding. The community is happy to help.

## Submitting a port

If you want to bring Catppuccin to a new application, follow the [port creation guide](/port-creation). It covers the full workflow from setting up your repository to publishing your port.

## Code of Conduct

All contributors are expected to follow the [Catppuccin Code of Conduct](https://github.com/catppuccin/.github/blob/main/CODE_OF_CONDUCT.md). Please read it before participating.
