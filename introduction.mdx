Catppuccin (/ˌkætpʊˈtʃiːn/) is a community-driven pastel theme that aims to be the middle ground between low and high-contrast themes. It consists of 4 soothing warm flavors with 26 eye-candy colors each, perfect for coding, designing, and much more.

## Design philosophy

Catppuccin is built on three guiding principles:

- **Colorful is better than colorless** — the colorfulness of something contributes to the distinction amongst the parts of that something, making it marginally easier to understand how things are structured.
- **There should be balance** — not too dull, not too bright. Suitability under various light conditions is a must.
- **Harmony is superior to dissonance** — vivacious colors must complement each other.

## Flavors

Catppuccin comes in four flavors. Three are dark-background variants and one is light, giving you a full spectrum from bright to deep.

<Note>
  All four flavors share the same 26 color names. Only the hex values differ between flavors. This means a port only needs to be written once and can be adapted to any flavor by swapping the palette values.
</Note>

## Ports

A **port** is an adaptation of the Catppuccin palette for a specific application, tool, or platform. Ports translate the 26 palette colors into whatever format the target app requires — whether that's a VS Code theme JSON file, a terminal color scheme, a CSS file, or a config snippet.

Ports exist for hundreds of applications, including:

- Code editors and IDEs (VS Code, Neovim, JetBrains, Emacs, Helix, Zed)
- Terminals (kitty, Alacritty, WezTerm, iTerm2, Ghostty)
- Browsers (Firefox, Chrome, Zen Browser)
- Desktop environments and window managers (KDE, Hyprland, i3)
- CLI tools (tmux, bat, fzf, Starship, lazygit)
- Programming language libraries (Rust, Python, Go, Java, Flutter)

## Sub-palettes

Every flavor is organized into two sub-palettes:

- **Monochromatic** — different shades of purple-gray and pale lavender, used for UI components like backgrounds, borders, and surfaces.
- **Analogous** — all the colors of the rainbow (Rosewater through Lavender), used for syntax highlighting and semantic color roles.

This structure means ports can map colors semantically rather than by exact value, ensuring consistent theming across very different applications.
