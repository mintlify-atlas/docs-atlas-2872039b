Since Catppuccin is available in 4 flavors, it can be a challenge to make 4 versions of a port. To help with that, the project includes a set of tools to make port creation easier. You'll find them all with instructions in the [catppuccin/toolbox](https://github.com/catppuccin/toolbox) repository.

## Key tools

<CardGroup cols={2}>
  <Card title="catppuccin/toolbox" icon="toolbox" href="https://github.com/catppuccin/toolbox">
    A collection of tools built to assist with creating Catppuccin ports, including catwalk and other utilities.
  </Card>
  <Card title="catwalk" icon="image" href="https://github.com/catppuccin/toolbox#-catwalk">
    Creates a layered screenshot of your port that combines all four flavors — Latte, Frappé, Macchiato, and Mocha — into one preview image.
  </Card>
  <Card title="catppuccin/template" icon="copy" href="https://github.com/catppuccin/template">
    A public template repository to use as a blueprint when creating a new port. Clone it to get started quickly with the right structure.
  </Card>
  <Card title="catppuccin/userstyles" icon="globe" href="https://github.com/catppuccin/userstyles">
    The home for all Catppuccin userstyles — ports for websites themed via Stylus. Submit your finished userstyle here via a pull request.
  </Card>
</CardGroup>

## Licensing

Any contribution will be published under the same licensing terms (MIT) as the project itself. However, there are [exceptions to this rule](https://github.com/search?q=org%3Acatppuccin+-license%3Amit).

If the application you're theming has licensing restrictions, or your work is a derivative of a stricter license (e.g. [cursors](https://github.com/catppuccin/cursors#info)), get in touch with the staff team before submitting.

## Maintenance

The staff team understands that submitting a port may feel overwhelming. **You are not expected to keep maintaining your port after it has been transferred to the organization.** Continued involvement is welcome and appreciated, but not required.

When you submit your port for transfer, think of it as offering it up for co-maintenance. You'll work alongside the staff team, whose focus is to maintain consistent quality and accessibility standards across all ports. The team will also provide feedback to improve your port and help resolve any technical issues.
