---

Follow this workflow when creating and submitting a port to the Catppuccin organization.

<Steps>
  <Step title="Have an idea for a port">
    Decide which app, tool, or website you want to theme with Catppuccin's palette.
  </Step>
  <Step title="Check if a draft already exists">
    Search the [Port Requests discussions](https://github.com/catppuccin/catppuccin/discussions/categories/port-requests) to see if someone has already started working on the same port.
  </Step>
  <Step title="Open a Port Request discussion">
    If no draft exists yet, open a [Port Request discussion](https://github.com/catppuccin/catppuccin/discussions/new?category=port-requests) on GitHub to let the community know you're working on it.
  </Step>
  <Step title="Work on a draft">
    You or someone else picks up the request and begins working on a draft port. Feel free to join the [Discord](https://discord.com/servers/catppuccin-907385605422448742) and share your progress there too.
  </Step>
  <Step title="Complete the draft">
    Finish the draft to a state you're happy with, following the [styling guidelines](/port-creation/styling).
  </Step>
  <Step title="Submit for review">
    How you submit depends on the type of port:

    - **Userstyle (Stylus-based):** Submit a [pull request to catppuccin/userstyles](https://github.com/catppuccin/userstyles/compare). You can find further information on how to contribute userstyles in the [userstyle creation docs](https://github.com/catppuccin/userstyles/blob/main/docs/userstyle-creation.md).
    - **All other ports:** Create a [Port Review issue](https://github.com/catppuccin/catppuccin/issues/new?assignees=&template=port-review.yml&title=Name+of+the+application%2Ftool%2Fwebsite%2Fetc.) in the main repository.
  </Step>
  <Step title="Review period">
    The [staff team](https://github.com/catppuccin/community/#current-structure) and community review your port and provide feedback. Address any comments and iterate as needed.
  </Step>
  <Step title="Port adoption">
    Once the review is complete, your port is adopted into the Catppuccin organization.
  </Step>
</Steps>

## FAQ

<AccordionGroup>
  <Accordion title="I have a port that is already themed and ready for review">
    Port reviews can be raised as an [issue](https://github.com/catppuccin/catppuccin/issues/new?assignees=&template=port-review.yml&title=Name+of+the+application%2Ftool%2Fwebsite%2Fetc.) since the port is already themed and ready to be reviewed by the [staff team](https://github.com/catppuccin/community/#current-structure).
  </Accordion>
  <Accordion title="I have a userstyle that is already finished and ready for review">
    Finished userstyles should be raised via a [pull request](https://github.com/catppuccin/userstyles/compare) to [catppuccin/userstyles](https://github.com/catppuccin/userstyles). You can find further information on how to contribute userstyles in the [userstyle creation docs](https://github.com/catppuccin/userstyles/blob/main/docs/userstyle-creation.md).
  </Accordion>
  <Accordion title="I have a request for a port and/or I've started working on it">
    Raise a [discussion](https://github.com/catppuccin/catppuccin/discussions/new?category=port-requests) under the main repository. The discussion will be transferred to an issue by the [staff team](https://github.com/catppuccin/community/#current-structure) once the port is deemed ready for review. Feel free to join the [Discord](https://discord.com/servers/catppuccin-907385605422448742) and share it there too.
  </Accordion>
  <Accordion title="What types of ports won't be accepted?">
    As a community-driven project, Catppuccin aims to keep a neutral environment for all users. **Contributions with a religious or political context are not accepted.** However, there is no issue with the palette being used in those contexts outside the organization.
  </Accordion>
</AccordionGroup>
